プロジェクト⑤

音声スライド生成システム

NoLangをベンチマークとした内製可能性の調査・検証

プロジェクト概要

「NoLang」のような、テキストやPDF、PowerPointから高品質な音声付き動画を生成できるAIサービスを、自社で開発する可能性を探ります。まずはNoLangの機能を調査・分析し、内製の実現可能性を検証したうえで、要件定義・技術仕様・設計書まで作成します。

🎯 ベンチマーク: NoLang

NoLangは、テキスト・PDF・PPTX・音声・動画の4入力から、AI音声・アバター・字幕付きの動画を自動生成するサービスです。本プロジェクトでは、NoLangの機能を参考にしつつ、自社教育コンテンツ事業に最適化したシステムの内製可能性を探ります。

📊 優先度: 比較的早期の判断が必要

インターンの方にリサーチいただき、実現可能性の壁打ちや実装を進めたいプロジェクトです。最優先ではありませんが、比較的はじめの方に判断してほしい案件として位置づけます。

NoLangで満たせそうな要件（公開情報ベース）

1. 入力形式（NoLangの4入力）

📥 対応入力
  • テキスト・Webページ
  • PDF・PPTX
  • 音声
  • 動画

NoLangは4つの入力形式から動画生成が可能と案内されています。

2. 音声合成の品質・表現

🔊 NoLang 4.0の特徴
  • 最新の音声合成モデル導入
  • 囁き声・抑揚など感情豊かな読み上げ
  • 100体以上のアバター（Live2D/3D）

3. 字幕連動（動画入力時）

📝 自動字幕機能

素材の動画ファイルをアップロードすると、字幕が自動追加され、内容に即した画像・アバターが挿入された動画を生成します。

4. 編集（テンプレ／要素調整）

✏️ カスタマイズ機能

テンプレート機能により、アバター・BGM・背景・テロップのフォントや位置などをカスタマイズ（編集）可能とされています。

5. API連携（NoLang API / OEM）

🔌 外部連携

NoLang APIは、外部サービス・自社システムに動画生成機能を組み込める旨が案内されています。api.no-lang.com/v1系のエンドポイント例も確認できます。

6. 商用・ライセンス注意

⚖️ ライセンス

FAQ抜粋では「通常の動画はCC-BY-SAで、コピーライト明記で商用利用可能」という旨が見えます。ただし運用時は必ず最新FAQ・規約で確定する必要があります。また、Webページ要約動画は権利関係の注意があるという解説もあります。

プロジェクト⑤の要求（整理）

目的

自社で研修動画／講座動画を効率生成する（教育コンテンツが事業の中核）

入力

  • テキスト
  • スライド画像
  • PDF
  • PowerPoint（可能なら）

出力

  • MP4動画
  • 埋め込み形式（プレイヤー／URL等）

言語

  • 当面: 日本語
  • 将来: 英語は視野（優先度は低い）

リサーチ観点（必須）

🔍 調査項目
  • 音声合成精度（最重要）
  • 字幕連動
  • アバター生成
  • エディター機能
  • API連携
  • 商用ライセンス
  • 内製可能性（技術的実現性・費用感）

戦略の選択肢（Buy / Hybrid / Build）

A. Buy（NoLangを使う）

✅ メリット
  • 最短で「入力→動画」まで到達しやすい
  • PDF/PPTX/音声/動画対応が明記済み
  • API/OEMの選択肢がある（社内ツールに組み込み可能）
⚠️ リスク
  • 権利・ライセンス条件（特に素材由来の著作権／クレジット表記）
  • 外部サービス依存（価格変更・サービス終了リスク）

B. Hybrid（当面NoLang、徐々に内製）

🔄 アプローチ
  • まずNoLang（またはNoLang API）で運用を成立させる
  • 並行で内製するのは「差別化ポイント」に限定
  • 例: 自社専用音声、固定アバター、LMS連携、監査ログ、教材テンプレ統一

C. Build（完全内製）

🛠️ 実現性

実現は可能だが、NoLang相当（字幕・アバター・編集UI・レンダリング・API・権利対応）まで全部やるとスコープが大きい。

📌 MVP（段階的アプローチ）

「音声品質最優先」のMVPなら、TTS＋字幕＋スライド動画合成（アバターは簡易）で段階的に作れる。

内製する場合の要件定義（MVP → 拡張）

MVP（まず作る価値が高い最小構成）

入力

PDF / PPTX / テキスト

台本生成

LLMでスライドごとにナレーション案を自動生成

音声合成

高品質TTS（候補: Google Cloud TTS, Azure Speech, OpenAI音声モデル等）

  • Google Cloud TTS: WaveNet/新世代HD音声
  • Azure Speech: ニューラル音声／言語対応
  • OpenAI: API向け音声モデル継続更新

字幕

台本からSRT/VTT生成（音声長に合わせて自動タイミング調整）

動画合成

スライド画像＋音声＋字幕をFFmpeg等でMP4化

管理

ジョブ投入／進捗／成果物ダウンロード

拡張（プロジェクト要件に近づける）

  • アバター（右下固定など）＋口パク（viseme）同期
    視覚的な案内役としてアバターを追加し、音声に合わせた口の動きを実現
  • 編集UI（スライド単位で台本・読み上げ差し替え）
    ユーザーが台本を編集し、再生成できるインターフェース
  • API公開（外部システム連携）
    LMSや社内システムとの連携API
  • 埋め込みプレイヤー
    顧客サイトへの埋め込み用プレイヤー
  • 日本語→英語の二言語対応
    将来的な多言語展開への対応

技術仕様（案：システム構成）

コンポーネント

Ingest Service

PDF/PPTX→スライド画像化＋テキスト抽出

Script Service

LLMで「スライド別ナレーション」生成

TTS Service

音声合成（プロバイダ差し替え可能に）

Subtitle Service

SRT/VTT生成＋タイミング補正

Render Service

FFmpegで合成（字幕＋アバターオーバーレイ）

Web App

ジョブ作成／編集／プレビュー／出力

Storage

素材（入力）・中間生成物・出力MP4

Audit/Log

生成履歴、誰が何を作ったか（法人向け必須）

API（最低限）

  • POST /jobs - 入力登録・生成開始
  • GET /jobs/{id} - 進捗・ステータス
  • GET /jobs/{id}/artifacts - MP4、SRT、台本など
  • POST /jobs/{id}/edit - 台本差し替え→部分再生成

インターンに依頼する検証タスク

以下の項目を調査・検証してもらいます。

1. NoLang調査

📋 調査項目
  • APIでどこまで制御できるか（テンプレ、音声、字幕、アバター、編集）
  • 商用利用条件（特にクレジット表記・入力素材の権利）
  • 料金体系・プラン
  • レート制限・利用制限
  • サービスレベル・サポート体制

2. 内製の場合の技術調査

🔬 検証項目
  • 日本語TTS品質の比較（Google / Azure / OpenAIなど）
  • レンダリング（字幕同期・アバター同期）の技術難易度と工数感
  • 段階開発の可能性（MVP → 拡張）
  • コスト試算（開発費 vs ランニングコスト vs NoLang利用料）

3. 実現可能性の壁打ち

💭 判断材料
  • Buy / Hybrid / Build のどれが最適か
  • 内製する場合のリスク・課題
  • NoLang利用の場合のリスク・課題
  • 推奨アプローチとその理由
