プロジェクト①

CanvaとAIを活用したコンテンツ制作

スライド生成と成形の自動化システム

📁 参照資料
📊 講座計画表（スプレッドシート）を開く

プロジェクト概要

講座計画表（スプレッドシート）に基づいて、AIを活用したスライド生成から成形までを効率的に行うフローを構築します。Deep Research、Gemini Canvas、Googleスライド、Canvaを組み合わせることで、属人性を減らし、規格化された高品質な教材スライドを作成できる環境を目指します。

🎯 現時点の優先範囲

このプロジェクトでは、まず「スライド生成と成形」までを業務化します。音声合成やアバター機能は次工程として検討しますが、当面はスライド原本の完成を目標とします。

主なフロー（手順）

以下の手順で、講座スライドの原本を作成します。

  1. 講座計画表でタイトル確認
    スプレッドシート上の講座計画表から、対象講座のタイトルを確認します。
  2. Deep Researchで講座骨子を決定
    スプレッドシートの「0→1」タブにあるプロンプトを使い、GPT（ChatGPT等）でDeep Researchを実行します。
    目的: 講座の骨子（どんな講座にするか等）を決める
    使用ツール: GPT（ChatGPT）
    入力: 「0→1」タブのプロンプト
  3. Gemini Canvasでスライド生成
    GeminiのCanvas機能を使用し、Deep Research結果とプラスαのプロンプト（補助プロンプト）を貼り付けてスライドを生成します。
    使用ツール: Gemini（Canvas機能）
    入力: Deep Research結果 + 補助プロンプト
  4. Googleスライドへインポート
    生成されたスライドをGoogleスライドにエクスポート/インポートします。
  5. Googleスライドで整形
    Googleスライド上で、はみ出た文字などを枠内に収める調整を行います。
    文字の大きさ調整、レイアウトの微調整など
  6. PDFで保存
    整形後のスライドをPDF形式で保存します。
  7. Canvaフォルダへ移動しデザイン統一
    保存したスライド（PDF/原本）を指定のCanvaフォルダに入れて、デザイン統一のために微調整します。
    ここまででスライド原本は完成

スライド配置の基準（動画化前提）

スライドは将来的に動画化することを前提としているため、以下のレイアウト基準を守ります。

📐 下側15%〜20%は字幕領域

動画の字幕が表示される領域のため、スライド要素を配置しません（空白として確保）。

🎭 右下付近はAIアバター用余白

AIアバター（キャラクター）を差し込みやすくするため、字幕領域と合わせて右下を空けます。

これらの余白調整は、主にCanvaで微修正する段階で意識します。

使用ツール

以下のツールを組み合わせてスライド制作を行います。

  • Googleスプレッドシート
    講座計画表の管理、プロンプトの保管
  • GPT（ChatGPT等）
    Deep Researchによる講座骨子の設計
  • Gemini Canvas
    スライドの自動生成
  • Googleスライド
    スライドの整形・調整
  • Canva
    デザイン統一・微調整

プロンプト（コピペ用）

以下のプロンプトは、業務で使用する定型文です。コピーボタンでコピーできます。

工程A：Deep Research用プロンプト

▼工程A：Deep Research（改修版 / 業務シーン補足＋スライド分割設計対応）
あなたは企業向けE-learning講座の「教材設計者／研究者」です。
これから提示する【講座情報】と【スライドタイトル一覧】をもとに、Deep Researchで根拠を確認しながら、
スライド作成に直結する"素材"を、ルールに従って作成してください。

# 0. 絶対ルール（最重要）
- 出典不明の断定は禁止。根拠が弱い主張は「根拠が弱い」と明記する。
- 抽象比喩（釣り・大海・竹・登山・戦国・宇宙など）での例え話は禁止。
- 例えは必ず「業務シーン型ミニケース（仕事の具体場面）」にする。
- 不要な専門用語の追加は禁止（必要な場合は短い定義を添える）。
- 受講者が行動に移せる形に落とす（知識だけで終わらせない）。

# 1. 目的（この仕事のゴール）
- スライド作成の"原稿材料"を作る（工程Bでそのままスライド化できる粒度）。
- 各タイトルを「原則2枚」に分割できるように、内容の塊（導入/要点/具体/注意/行動）を揃える。
- 補足枠に使える「業務シーン型ミニケース」を、短く用意する。

# 2. 講座情報（ここを埋める）
【講座テーマ】{COURSE_THEME}
【受講者像】{AUDIENCE}（例：製造・物流現場／2年目社員／外国人比率など）
【学習ゴール】{LEARNING_GOALS}（行動ベースで3〜5個）
【想定尺】{DURATION}（例：30分、45分など）
【トーン】{TONE}（例：前向き、厳しめ、現場寄り 等）
【ユニット構成】{UNIT_PLAN}（例：全4ユニット、各22タイトル 等）
【NG】{NG_LIST}（例：煽りすぎ、専門用語過多、長文など）

# 3. 入力：スライドタイトル一覧（ここを貼る）
- 形式：ユニット番号 / タイトル番号 / タイトル
{SLIDE_TITLES}

# 4. 調査要件（Deep Researchのやり方）
- 各タイトルごとに「何が正しいか」を確認し、根拠ソースを必ず提示する。
- ソース優先順位：
  1) 公的機関・学会・大学・政府統計
  2) 査読論文／書籍（出版社・著者明記）
  3) 信頼性の高い企業レポート（調査方法が明記されているもの）
- "よくある言い回し"や出典不明の定説（例：学習ピラミッド等）は、エビデンスの有無を明確にし、扱い方を提案する。
- 例：「根拠が弱い／誤解が多い→使うなら注意書き／別概念に置換」など。
- 最新性が重要な内容（制度、統計、用語定義等）は、必ず一次情報で確認する（確認日も記載）。

# 5. 出力フォーマット（タイトルごとに必ずこの構造）
※各タイトルは「通し番号（そのユニット内で1から連番）」を付けて出力する。
※この通し番号は工程Bのスライドタイトルにそのまま使用する（例：1. ～ / 1. ～（続き））

【No】{n}（ユニット内の通し番号）
【Title】{そのままのタイトル}

A) 受講者に伝えるべき結論（1文）
B) キーメッセージ（最大3点、短く）
C) 業務シーン型ミニケース（仕事の具体場面を1つ）
   - 抽象比喩は禁止
   - 1〜2文（状況→行動→結果）で完結
D) 誤解・落とし穴（1〜2点）
E) 行動につなぐ一言（1文：次に何をするか）
F) クイズ（1問：選択式 or ○×）＋正解＋短い解説
G) ミニワーク（30秒〜2分でできる問い）※可能なら
H) 2枚分割の配分案（工程Bで使うために必須）
   - 1/2枚目（"なぜ/何"）：このスライドに入れる要素
   - 2/2枚目（"どうやって/具体"）：このスライドに入れる要素
I) 補足枠候補（任意）
   - 使うなら「補足：」で始める短文を1つ
   - 最大2行想定
   - 必ず業務シーン型（抽象比喩禁止）
J) 根拠ソース（2〜5件）
   - 形式：タイトル／著者・機関／年／リンク（URL）／一言要約／確認日（必要なら）

工程B：Gemini Canvas用プロンプト

▼工程B：Gemini Canvas（改修版 / タイトル連番＋業務シーン補足＋安全域統合）
あなたはE-learning講座のスライド構成・デザインのプロフェッショナルです。
以下の【Deep Research結果】と【スライドタイトル一覧】を元に、スライドをゼロから生成してください。
（Gemini Canvasでスライド化する前提）

# 0. 絶対ルール（最重要）
- スライド内に「文字入り画像」を作らない（画像・図形を使う場合も文字は入れない）。
- 抽象比喩（釣り・大海・竹・登山・戦国・宇宙など）での補足は禁止。補足は業務シーン型のみ。
- 読みやすさ最優先。1枚に詰め込みすぎない。

# 1. 目的（この仕事のゴール）
- 工程Aの素材を、字幕・アバター前提で読みやすいスライドに落とす。
- 1タイトルを原則2枚に分け、学習テンポが良い流れにする。
- もともと短く美しく収まる場合は、SMART_FITで1枚運用も可。

# 2. 出力モード（どちらかを採用）
【MODE】= {STRICT_2P / SMART_FIT}

- STRICT_2P：
  - 各タイトルを必ず2枚にする（総枚数を厳密に合わせる）
  - 「(1)(2)／(3)だけ」等の不格好分割は禁止。密度を均等化する。
- SMART_FIT：
  - 原則2枚だが、1枚でバチッと収まるなら1枚でも良い（見た目最優先）

# 3. タイトル表示ルール（必須：通し番号＋（続き））
- 各スライドは必ずタイトルを最上部先頭に大きく配置する。
- タイトルは必ず「通し番号. タイトル」で統一する（通し番号は工程Aの【No】を使用）。
- 1タイトルを2枚に分ける場合：
  - 1/2枚目：{n}. {タイトル}
  - 2/2枚目：{n}. {タイトル}（続き）
- 1枚のままの場合は「（続き）」は付けない。

# 4. レイアウト安全域（必須）
- 下部10%は必ず空ける（字幕・UI安全域）。本文テキスト・補足ともに侵入禁止。
- 右下を必ず空けるルールは設けない（右側に画像を置く運用を優先）。
- 重なりの許容：
  - 本文テキスト × 画像/アバター：重なり禁止（最優先）
  - 画像 × アバター：重なり許可
- 本文テキストは左側の本文エリアに固定し、右側はビジュアルエリアとして確保する。

# 5. 文字量の上限（行数ではなく密度で管理）
- 1枚に載せる文字は「一目で追える密度」を上限とする。
- 目安（推奨）：
  - 箇条書き中心：5〜7行程度
  - 文章中心：4〜6行程度
  ※左右2カラム等で視認性が保てるなら例外可。
- 上限を超えそうなら、2枚に再配分して密度を均等化する。

# 6. 分割ルール（見栄えを崩さないための具体）
- 分割の基本は「意味の塊」で切る：
  - 定義→理由→方法→まとめ
  - 対比（A vs B）
  - 手順（Step）・観点（定量/定性）・対象（上司/同僚/後輩）など
- 箇条書き（ポイント3つ等）の扱い：
  - 1枚で読みやすいなら1枚に集約（SMART_FITなら推奨）。
  - どうしても2枚にする場合（STRICT_2P）：
    - 「(1)(2)／(3)だけ」にならないよう、文章も合わせて配分し密度を均等化する。

# 7. 補足枠（任意・最小 / 業務シーン型のみ）
- 工程Aの「補足枠候補」を、必要な場合に限り「補足枠」として追加してよい。
- 補足枠の制約：
  - 必ず「補足：」で開始し、本文と別ブロック
  - 1スライドにつき最大2行（または短い箇条書き1点）
  - 下部10%安全域には置かない
  - 原則2/2枚目に置く
- 禁止：
  - 抽象比喩、長文解説、煽り、余計な専門用語、文字量の増加

# 8. レイアウト型（いずれかを選ぶ）
A) タイトル＋短い導入（上）＋要点（中）＋余白（下）
B) タイトル＋左右2カラム（対比・分類）
C) タイトル＋上段に定義/結論＋下段に箇条書き
D) タイトル＋箇条書き中心（文字が少ない時のみ）

# 9. 生成するスライド構成（各ユニット）
- 表紙：ユニット名、講座名（必要なら）、サブタイトル（短く）
- 本文：タイトル一覧に従って生成
- まとめ：ユニット末尾に1枚（任意：SMART_FITでは入れて良い。STRICT_2Pで枚数固定なら条件に従う）

# 10. 出力フォーマット（必ずこの形）
## Unit情報
- Unit名：{UNIT_NAME}
- MODE：{MODE}
- 目標枚数：表紙＋{TARGET_BODY_SLIDES}枚（STRICT_2Pの場合は厳守）

## Slides
各スライドごとに下記6項目：
1) Slide ID：{UNIT}-{番号}（例：U1-01）
2) Slide種別：{表紙 / 本文}
3) スライドタイトル：{n}. {タイトル}（2/2枚目は同じnで（続き））
4) 本文テキスト：工程AのA〜Eを中心に、読みやすい塊で配置（※工程Aの主張を歪めない）
5) 補足枠（任意）：補足が無い場合は「なし」
6) レイアウト指定：A/B/C/D＋配置メモ（左本文／右ビジュアル／下部10%空白など）

# 11. 入力（ここを貼る）
【スライドタイトル一覧】
{SLIDE_TITLES}

【Deep Research結果（工程Aの出力をそのまま貼る）】
{DEEP_RESEARCH_RESULT}

# 12. 最終セルフチェック（出力前に必ず満たす）
- 下部10%安全域が確保されている（本文・補足とも侵入していない）
- 本文が画像/アバターと重なっていない
- スライドタイトルが「通し番号. タイトル」で統一されている
- 2枚分割時、2/2枚目タイトルに必ず（続き）が付いている（番号は同一）
- 過密スライドが無い（必要なら分割）
- 補足は最小で、業務シーン型のみ（抽象比喩が混ざっていない）
- 文字入り画像を作る指示が含まれていない
